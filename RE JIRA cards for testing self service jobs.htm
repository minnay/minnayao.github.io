In the last couple of months, we have had doubts and questions about “when to push playbook changes to the prod Ansible host  (currently btcrd1ansi6)?” “How to push playbook changes to the prod Ansible host?”

 

We have all agreed that we should treat the Ansible prod just like a prod Java server, which means to push any changes to the prod, we need to go through thorough testing (automated and/or manual). Currently our automated tests only cover the creation and termination of AWS instances; we rely completely on manual testing to cover other important aspect of the platform offering, for example, self service.

 

Without automated tests for self service, there are a few pressing problems:

-          Having doubts and low confidence whether we can pull playbook changes onto the prod Ansible; unless the end to end tests have all passed, and manually tested all of the self service jobs .

-          Pulling changes to the prod Ansible is manual. Our customers would not enjoy the latest features of self-service immediately. PLACOE is delaying in offering continuous delivery.
